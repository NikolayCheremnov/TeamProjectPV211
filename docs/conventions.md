# Конвенции командного проекта ПВ211

### Правила написания кода

Для успешной разработки необходимо принять следующие правила написании и реализации кода:
    
1. Общие стиль кода должен соответствовать общепринятым [конвенциям](https://learn.microsoft.com/en-us/dotnet/csharp/fundamentals/coding-style/coding-conventions) разработки на языке C#.
2. Имена переменных должны быть понятными, содержательными. Запрещено использовать транслит, переменные именуются только на английском языке. Из названия переменной должно быть ясно, что в ней хранится.

### Правила написания коммитов

1. Коммиты писать на английском языке
2. Текст коммита должен отражать действие, выполненное в рамках данного коммита
3. Использовать инфинитивы в коммитах
4. Формулировать коммиты кратко, понятно
5. Начинать коммит с большой буквы, в конце точку не ставить

Примеры:

- Add registration sign entity - хорошо
- Added registration sign entity - плохо - не инфинитив
- add registration sign entity - плохо - с маленькой буквы
- registration sign entity - плохо - отсутствует информацию о действии

### Алгоритм работы над задачей:

1. Выбрать задачу и взять ее в работу. Перевести статус на доске задач в In Progress
2. Создать отдельную ветку для задачи выбрав ей подходящее имя, которое резюмирует действия, выполняющиеся в рамках задачи (отталкиваться от формулировки задачи). Слова в имени ветки разделять символом `-`.
3. Выполняется работа над задачей. Делаются коммиты.
4. Создать pull request (PR) (запрос на вливание ветки с задачей в главную ветку):
    1) Выполнить push ветки - отправить ветку на удаленный репозиторий
    2) Создать новый PR, выбрать свою ветку в качестве источника, master в качестве целевой ветки. Добавить заголовок и описание PR на русском языке. Назначить себя ответственным за этот PR. Добавить в описание ссылку на задачу.
    3) Пройти через ревью (предварительно перевести задачу в статус Review) и получить одобрение на выполнение слияния. Так же исключить конфликты связанные с главной веткой. Для исключения конфликтов, связанных с главной веткой выполнить сначала операцию rebase для главной ветки. Если в ветке было несколько коммитов, то рекомендуется выполнить операцию squash для них, сделав из них один коммит.
5. Выполнить merge. Удалить ветку, закрыть задачу.
